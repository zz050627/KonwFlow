# 代码中的隐式规则与约定

## 1. 命名约定

### 1.1 包命名规则

```
com.nageoffer.ai.ragent.{module}.{layer}.{feature}

示例：
com.nageoffer.ai.ragent.rag.service.impl        # RAG模块的服务实现
com.nageoffer.ai.ragent.knowledge.dao.mapper    # 知识库模块的Mapper
com.nageoffer.ai.ragent.framework.context       # 框架模块的上下文
```

**层级约定**：
- `controller`：控制器层（REST API）
- `service`：服务层（业务逻辑）
- `service.impl`：服务实现
- `dao`：数据访问层
- `dao.entity`：实体类（DO）
- `dao.mapper`：MyBatis Mapper接口
- `dto`：数据传输对象
- `core`：核心组件
- `config`：配置类
- `aop`：AOP切面

### 1.2 类命名规则

| 类型 | 后缀 | 示例 | 说明 |
|------|------|------|------|
| 实体类 | `DO` | `KnowledgeBaseDO` | Database Object |
| 数据传输对象 | `Request`/`Response` | `ChatRequest` | DTO |
| 值对象 | `VO` | `ChunkVO` | View Object |
| Mapper接口 | `Mapper` | `KnowledgeBaseMapper` | MyBatis接口 |
| 服务接口 | `Service` | `RAGChatService` | 业务接口 |
| 服务实现 | `ServiceImpl` | `RAGChatServiceImpl` | 业务实现 |
| 控制器 | `Controller` | `RAGChatController` | REST控制器 |
| 配置类 | `Config` | `ThreadPoolExecutorConfig` | 配置类 |
| 切面类 | `Aspect` | `ChatRateLimitAspect` | AOP切面 |
| 工具类 | `Util` | `RedisKeyUtil` | 工具类 |
| 常量类 | `Constant` | `RAGConstant` | 常量定义 |

## 4. 异常处理规则

### 4.1 异常体系

```
AbstractException (抽象基类)
├── ClientException (4xx，客户端错误)
│   ├── 参数校验失败
│   ├── 资源不存在
│   └── 权限不足
└── ServiceException (5xx，服务端错误)
    ├── 数据库异常
    ├── 外部服务调用失败
    └── 系统内部错误
```

### 4.2 异常抛出约定

```java
// 客户端错误（用户输入问题）
if (StrUtil.isBlank(knowledgeBaseId)) {
    throw new ClientException("知识库ID不能为空");
}

if (knowledgeBase == null) {
    throw new ClientException("知识库不存在");
}

// 服务端错误（系统问题）
try {
    milvusClient.search(...);
} catch (Exception e) {
    throw new ServiceException("向量检索失败", e);
}
```

### 4.3 全局异常处理

```java
@RestControllerAdvice
public class GlobalExceptionHandler {
    
    @ExceptionHandler(ClientException.class)
    public Result<?> handleClientException(ClientException e) {
        return Result.failure(e.getErrorCode(), e.getMessage());
    }
    
    @ExceptionHandler(ServiceException.class)
    public Result<?> handleServiceException(ServiceException e) {
        log.error("服务异常", e);
        return Result.failure("系统繁忙，请稍后重试");
    }
}
```

### 4.4 异常日志规则

```java
// 预期异常：warn级别，不打印堆栈
log.warn("文档处理失败：{}", documentId);

// 非预期异常：error级别，打印完整堆栈
log.error("向量检索异常", e);

// 关键业务异常：error级别 + 告警
log.error("支付失败，订单ID：{}", orderId, e);
alertService.sendAlert("支付异常", orderId);
```

## 5. 日志规则

### 5.1 日志级别使用

| 级别 | 用途 | 示例 |
|------|------|------|
| `ERROR` | 系统错误，需要立即处理 | 数据库连接失败、外部服务不可用 |
| `WARN` | 警告信息，可能影响功能 | 配置缺失、降级处理、重试 |
| `INFO` | 关键业务流程 | 用户登录、文档上传、对话开始 |
| `DEBUG` | 详细调试信息 | 方法入参、中间结果、SQL语句 |
| `TRACE` | 最详细的追踪信息 | 循环内部、变量变化 |

```java
// 业务流程日志：包含关键标识
```

## 7. 事务、并发、重试、幂等规则

### 7.1 事务规则

**约定**：Service层方法添加 `@Transactional`

```java
@Service
@RequiredArgsConstructor
public class KnowledgeDocumentServiceImpl {
    
    @Transactional(rollbackFor = Exception.class)
    public void uploadDocument(MultipartFile file, String knowledgeBaseId) {
        // 1. 上传文件
        String fileUrl = rustFsClient.upload(file);
        
        // 2. 插入文档记录
        documentMapper.insert(document);
        
        // 3. 更新知识库统计
        knowledgeBaseMapper.incrementDocumentCount(knowledgeBaseId);
        
        // 任何异常都会回滚
    }
}
```

**注意事项**：
- ❌ 不要在事务内调用外部HTTP服务（可能超时）
- ❌ 不要在事务内执行长时间操作
- ✅ 事务尽量短小，只包含数据库操作

### 7.2 并发控制

**约定1：分布式锁**
```java
// 使用Redisson分布式锁
RLock lock = redissonClient.getLock("rag:document:scan");
if (lock.tryLock(10, 900, TimeUnit.SECONDS)) {
    try {
        // 临界区代码
        scanAndProcessDocuments();
    } finally {
        lock.unlock();
    }
}
```

**约定2：分布式信号量**

### 7.3 重试规则

**约定**：使用 Spring Retry

```java
@Service
public class ExternalApiService {
    
    @Retryable(
        value = {HttpClientErrorException.class},
        maxAttempts = 3,
        backoff = @Backoff(delay = 1000, multiplier = 2)
    )
    public String callExternalApi(String url) {
        // 失败自动重试，最多3次，延迟1s/2s/4s
        return restTemplate.getForObject(url, String.class);
    }
    
    @Recover
    public String recover(HttpClientErrorException e, String url) {
        // 重试失败后的降级处理
        log.error("外部API调用失败，URL：{}", url, e);
        return "默认值";
    }
}
```

### 7.4 幂等规则

**约定**：使用 `@IdempotentSubmit` 注解

```java
@Service
public class KnowledgeDocumentServiceImpl {
    
    @IdempotentSubmit(key = "#request.requestId", expireSeconds = 5)
    public void uploadDocument(UploadRequest request) {
        // 5秒内相同requestId的请求会被拒绝
    }
}
```

**实现原理**：
```java
@Around("@annotation(IdempotentSubmit)")
public Object around(ProceedingJoinPoint joinPoint, IdempotentSubmit annotation) {
    String key = parseKey(annotation.key(), joinPoint);
    String redisKey = "rag:idempotent:" + key;
    
    // 尝试设置Redis Key（NX：不存在才设置）
    Boolean success = redisTemplate.opsForValue()
        .setIfAbsent(redisKey, "1", annotation.expireSeconds(), TimeUnit.SECONDS);
    
    if (Boolean.FALSE.equals(success)) {
        throw new ClientException("请勿重复提交");
    }
    
    return joinPoint.proceed();
}
```

## 8. 隐藏的技术约定

### 8.1 线程池上下文传递

**约定**：所有线程池必须用 `TtlExecutors` 包装

```java
// ❌ 错误：普通线程池，UserContext会丢失
ExecutorService executor = Executors.newFixedThreadPool(4);

// ✅ 正确：TTL包装，UserContext自动传递
ExecutorService executor = TtlExecutors.getTtlExecutorService(
    Executors.newFixedThreadPool(4)
);
```

**原因**：`UserContext` 使用 `TransmittableThreadLocal`，需要 TTL 支持。

### 8.2 MyBatis-Plus自动填充

**约定**：实体类字段自动填充

```java
public class KnowledgeBaseDO {
    
    @TableField(fill = FieldFill.INSERT)
    private LocalDateTime createTime;  // 插入时自动填充
    
    @TableField(fill = FieldFill.INSERT_UPDATE)
    private LocalDateTime updateTime;  // 插入和更新时自动填充
    
    @TableLogic
    private Integer delFlag;  // 逻辑删除标记

```

**配置**：
```java
@Component
public class MyMetaObjectHandler implements MetaObjectHandler {
    
    @Override

        this.strictInsertFill(metaObject, "createTime", LocalDateTime.class, LocalDateTime.now());
        this.strictInsertFill(metaObject, "updateTime", LocalDateTime.class, LocalDateTime.now());
    }
    
    @Override
    public void updateFill(MetaObject metaObject) {
        this.strictUpdateFill(metaObject, "updateTime", LocalDateTime.class, LocalDateTime.now());
    }
}
```
