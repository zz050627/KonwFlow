# 核心模块详解

## 1. RAG 对话模块（rag/）

### 模块职责
实现完整的 RAG 对话流程，包括记忆管理、查询改写、意图识别、多通道检索、Prompt 组装、流式输出。

### 关键文件

#### 1.1 RAGChatServiceImpl.java
**路径**：`bootstrap/src/main/java/com/nageoffer/ai/knowflow/rag/service/impl/RAGChatServiceImpl.java`

**职责**：核心流程编排器，协调所有子组件完成对话

**核心方法**：
```java
@ChatRateLimit  // 分布式限流
public void streamChat(String question, String conversationId, 
                       Boolean deepThinking, String modelId, SseEmitter emitter) {
    // 1. 记忆加载
    List<ChatMessage> history = memoryService.loadAndAppend(conversationId, userId, question);
    
    // 2. 查询改写（上下文补全 + 拆分子问题）
    RewriteResult rewriteResult = queryRewriteService.rewriteWithSplit(question, history);
    
    // 3. 意图识别
    List<SubQuestionIntent> subIntents = intentResolver.resolve(rewriteResult);
    
    // 4. 歧义检测（低置信度触发澄清）
    GuidanceDecision guidanceDecision = guidanceService.detectAmbiguity(rewrittenQuestion, subIntents);
    if (guidanceDecision.isPrompt()) {
        callback.onContent(guidanceDecision.getPrompt());
        return;
    }
    
    // 5. 多通道检索
    RetrievalContext retrievalContext = retrievalEngine.retrieve(subIntents, topK);
    
    // 6. MCP 工具调用（如果意图是工具类型）
    if (hasToolIntent) {
        mcpService.executeTool(...);
    }
    
    // 7. Prompt 组装
    String finalPrompt = promptBuilder.buildPrompt(chunks, history, systemPrompt);
    
    // 8. 流式生成
    llmService.streamChat(finalPrompt, modelId, callback);
}
```

**与其他模块关系**：
- 依赖 `ConversationMemoryService`（记忆）
- 依赖 `QueryRewriteService`（改写）
- 依赖 `IntentResolver`（意图）
- 依赖 `RetrievalEngine`（检索）
- 依赖 `LLMService`（生成）

#### 1.2 IntentResolver.java
**路径**：`bootstrap/src/main/java/com/nageoffer/ai/knowflow/rag/core/intent/IntentResolver.java`

**职责**：树状多级意图识别（领域 → 类别 → 主题）

**核心逻辑**：
```java
public List<SubQuestionIntent> resolve(RewriteResult rewriteResult) {
    List<SubQuestionIntent> results = new ArrayList<>();
    
    // 对每个子问题进行意图识别
    for (SubQuestion subQ : rewriteResult.subQuestions()) {
        // 1. 领域分类（技术/产品/通用）
        String domain = classifyDomain(subQ.question());
        
        // 2. 类别分类（根据领域进一步细分）
        String category = classifyCategory(subQ.question(), domain);
        
        // 3. 主题提取（具体关注点）
        String topic = extractTopic(subQ.question());
        
        // 4. 置信度计算
        double confidence = calculateConfidence(domain, category, topic);
        
        results.add(new SubQuestionIntent(subQ, domain, category, topic, confidence));
    }
    
    return results;
}
```

#### 1.3 RetrievalEngine.java
**路径**：`bootstrap/src/main/java/com/nageoffer/ai/knowflow/rag/core/retrieve/RetrievalEngine.java`

**职责**：多通道并行检索 + 后处理

**核心流程**：
```java
public RetrievalContext retrieve(List<SubQuestionIntent> intents, int topK) {
    // 1. 选择检索通道（根据意图置信度）
    List<SearchChannel> channels = selectChannels(intents);
    
    // 2. 并行检索
    List<CompletableFuture<List<Chunk>>> futures = channels.stream()
        .map(channel -> CompletableFuture.supplyAsync(
            () -> channel.search(intents, topK),
            multiChannelRetrievalExecutor
        ))
        .toList();
    
    // 3. 合并结果
    List<Chunk> allChunks = futures.stream()
        .map(CompletableFuture::join)
        .flatMap(List::stream)
        .toList();
    
    // 4. 后处理（去重 → 重排）
    List<Chunk> processed = postProcess(allChunks);
    
    // 5. 截断到 topK
    return new RetrievalContext(processed.subList(0, Math.min(topK, processed.size())));
}
```

**通道选择策略**：
- 意图置信度 >= 0.6：只用意图导向检索
- 意图置信度 < 0.6：意图 + 关键词 + 全局向量（三通道并行）

#### 1.4 ConversationMemoryService.java
**路径**：`bootstrap/src/main/java/com/nageoffer/ai/knowflow/rag/core/memory/ConversationMemoryService.java`

**职责**：会话记忆管理（滑动窗口 + 自动总结）

**核心方法**：
```java
public List<ChatMessage> loadAndAppend(String conversationId, String userId, ChatMessage newMessage) {
    // 1. 从 Redis 加载历史
    ConversationMemory memory = redisTemplate.opsForValue().get(buildKey(conversationId));
    
    // 2. 追加新消息
    memory.getMessages().add(newMessage);
    
    // 3. 滑动窗口（保留最近N轮）
    if (memory.getMessages().size() > historyKeepTurns * 2) {
        memory.setMessages(memory.getMessages().subList(
            memory.getMessages().size() - historyKeepTurns * 2,
            memory.getMessages().size()
        ));
    }
    
    // 4. 检查是否需要总结
    if (memory.getMessages().size() >= summaryStartTurns * 2 && memory.getSummary() == null) {
        CompletableFuture.runAsync(() -> {
            String summary = llmService.summarize(memory.getMessages());
            memory.setSummary(summary);
            redisTemplate.opsForValue().set(buildKey(conversationId), memory, ttlMinutes, TimeUnit.MINUTES);
        }, memorySummarizationExecutor);
    }
    
    // 5. 更新 Redis（设置 TTL）
    redisTemplate.opsForValue().set(buildKey(conversationId), memory, ttlMinutes, TimeUnit.MINUTES);
    
    return memory.getMessages();
}
```

**核心方法**：
```java
@IdempotentSubmit  // 幂等控制
public void uploadDocument(MultipartFile file, String knowledgeBaseId) {
    // 1. 获取分布式信号量（限制并发上传数）
    RSemaphore semaphore = redissonClient.getSemaphore("rag:document:upload");
    boolean acquired = semaphore.tryAcquire(maxWaitSeconds, TimeUnit.SECONDS);
    if (!acquired) {
        throw new ServiceException("系统繁忙，请稍后重试");
    }
    
    try {
        // 2. 上传到 RustFS
        String fileUrl = rustFsClient.upload(file);
        
        // 3. 创建文档记录（状态：PENDING）
        KnowledgeDocumentDO document = new KnowledgeDocumentDO();
        document.setKnowledgeBaseId(knowledgeBaseId);
        document.setFileName(file.getOriginalFilename());
        document.setFileUrl(fileUrl);
        document.setStatus(DocumentStatus.PENDING);
        documentMapper.insert(document);
        
        // 4. 发送 MQ 消息触发异步处理
        rocketMQTemplate.asyncSend("document-ingestion-topic", 
            new DocumentIngestionMessage(document.getId()),
            new SendCallback() { ... }
        );
        
    } finally {
        semaphore.release();
    }
}
```

**状态流转**：
```
PENDING → RUNNING → COMPLETED
   ↓         ↓
 FAILED   TIMEOUT
```

#### 2.2 KnowledgeDocumentScheduler.java
**路径**：`bootstrap/src/main/java/com/nageoffer/ai/knowflow/knowledge/scheduler/KnowledgeDocumentScheduler.java`

**职责**：定时扫描待处理文档，恢复超时任务

**核心逻辑**：
```java
@Scheduled(fixedDelayString = "${rag.knowledge.schedule.scan-delay-ms:10000}")
public void scanAndProcessDocuments() {
    // 1. 获取分布式锁（防止多实例重复执行）
    RLock lock = redissonClient.getLock("rag:document:scan");
    if (!lock.tryLock()) {
        return;
    }
    
    try {
        // 2. 查询 PENDING 文档
        List<KnowledgeDocumentDO> pendingDocs = documentMapper.selectList(
            new LambdaQueryWrapper<KnowledgeDocumentDO>()
                .eq(KnowledgeDocumentDO::getStatus, DocumentStatus.PENDING)
                .last("LIMIT " + batchSize)
        );
        
        // 3. 查询 RUNNING 超时文档（超过5分钟）
        List<KnowledgeDocumentDO> timeoutDocs = documentMapper.selectList(
            new LambdaQueryWrapper<KnowledgeDocumentDO>()
                .eq(KnowledgeDocumentDO::getStatus, DocumentStatus.RUNNING)
                .lt(KnowledgeDocumentDO::getUpdateTime, 
                    LocalDateTime.now().minusMinutes(5))
        );
        
        // 4. 重置超时文档状态为 PENDING
        timeoutDocs.forEach(doc -> {
            doc.setStatus(DocumentStatus.PENDING);
            documentMapper.updateById(doc);
        });
        
        // 5. 重新发送 MQ 消息
        Stream.concat(pendingDocs.stream(), timeoutDocs.stream())
            .forEach(doc -> rocketMQTemplate.asyncSend(...));
        
    } finally {
        lock.unlock();
    }
}
```

## 3. 文档摄取模块（ingestion/）

### 模块职责
异步处理文档：解析 → 分块 → 向量化 → 存储。

### 关键文件

#### 3.1 DocumentIngestionConsumer.java
**路径**：`bootstrap/src/main/java/com/nageoffer/ai/knowflow/ingestion/consumer/DocumentIngestionConsumer.java`

**职责**：RocketMQ 消费者，处理文档摄取

**核心流程**：
```java
@RocketMQMessageListener(topic = "document-ingestion-topic", consumerGroup = "knowflow-consumer")
public class DocumentIngestionConsumer implements RocketMQListener<DocumentIngestionMessage> {
    
    @Override
    public void onMessage(DocumentIngestionMessage message) {
        String documentId = message.getDocumentId();
        
        try {
            // 1. 更新状态为 RUNNING
            documentService.updateStatus(documentId, DocumentStatus.RUNNING);
            
            // 2. 下载文件
            byte[] fileBytes = rustFsClient.download(document.getFileUrl());
            
            // 3. 解析文档（Tika）
            String content = documentParser.parse(fileBytes, document.getFileName());
            
            // 4. 文本分块
            List<TextChunk> chunks = chunker.chunk(content, chunkSize, chunkOverlap);
            
            // 5. 批量向量化
            List<float[]> vectors = embeddingService.batchEmbed(
                chunks.stream().map(TextChunk::getContent).toList()
            );
            
            // 6. 提取关键词（每个分块）
            List<String> keywords = chunks.stream()
                .map(chunk -> keywordExtractor.extract(chunk.getContent()))
                .flatMap(List::stream)
                .toList();
            
            // 7. 存入向量数据库
            vectorStoreService.batchInsert(document.getKnowledgeBaseId(), chunks, vectors, keywords);
            
            // 8. 存入关系数据库
            chunkService.batchInsert(chunks);
            
            // 9. 更新状态为 COMPLETED
            documentService.updateStatus(documentId, DocumentStatus.COMPLETED);
            
        } catch (Exception e) {
            log.error("文档处理失败：{}", documentId, e);
            documentService.updateStatus(documentId, DocumentStatus.FAILED);
        }
    }
}
```

#### 4.1 RoutingLLMService.java
**路径**：`infra-ai/src/main/java/com/nageoffer/ai/knowflow/infra/chat/RoutingLLMService.java`

**职责**：多模型路由 + 故障转移

**核心逻辑**：
```java
public void streamChat(String prompt, String modelId, StreamCallback callback) {
    // 1. 获取候选模型列表（按优先级排序）
    List<ModelCandidate> candidates = modelSelector.selectCandidates(modelId);
    
    // 2. 依次尝试
    for (ModelCandidate candidate : candidates) {
        if (!candidate.isEnabled()) continue;
        
        try {
            // 3. 获取对应的客户端
            ChatClient client = getClient(candidate.getProvider());
            
            // 4. 首包探测（等待第一个token）
            AtomicBoolean firstTokenReceived = new AtomicBoolean(false);
            StreamCallback wrappedCallback = new StreamCallback() {
                @Override
                public void onContent(String content) {
                    firstTokenReceived.set(true);
                    callback.onContent(content);
                }
            };
            
            // 5. 发起请求
            client.streamChat(prompt, candidate.getModel(), wrappedCallback);
            
            // 6. 等待首包（超时3秒）
            if (waitForFirstToken(firstTokenReceived, 3000)) {
                return;  // 成功，直接返回
            }
            
        } catch (Exception e) {
            log.warn("模型 {} 调用失败，尝试下一个", candidate.getId(), e);
            modelSelector.recordFailure(candidate.getId());
        }
    }
    
    throw new ServiceException("所有模型均不可用");
}
```

```java
public List<String> extract(String text) {
    String prompt = String.format(
        "从以下文本中提取3-5个关键词，用逗号分隔：\n\n%s",
        text
    );
    
    String response = ollamaChatClient.chat(prompt, keywordModelId);
    
    // 解析关键词
    return Arrays.stream(response.split("[,，]"))
        .map(String::trim)
        .filter(s -> !s.isEmpty())
        .limit(maxKeywords)
        .toList();
}
```

## 模块间依赖关系

```
bootstrap (业务)
    ↓ 依赖
infra-ai (AI抽象)
    ↓ 依赖
framework (基础设施)
```

**设计原则**：
- framework 无业务逻辑，可被任何模块依赖
- infra-ai 只依赖 framework，不依赖 bootstrap
- bootstrap 是唯一的业务模块，依赖所有基础模块
